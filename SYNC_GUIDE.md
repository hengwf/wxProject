## 🚀 快速开始

### 1️⃣ 启动后台服务

```bash
# 进入后台目录
cd admin

# 安装依赖（如果还没安装）
npm install

# 启动服务
npm start
```

后台会在 `http://localhost:3000` 启动

---

### 2️⃣ 配置小程序

1. 打开 `utils/questionManager.js`
2. 修改服务器地址（如果你的后台不是 localhost）：

```javascript
// 服务器地址配置 - 请根据你的实际情况修改
const SERVER_URL = 'http://你的服务器地址:3000'
```

---

### 3️⃣ 小程序开发工具配置

**重要！** 微信开发者工具需要配置以下内容：

1. 打开微信开发者工具
2. 点击右上角「详情」
3. 勾选「不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书」

⚠️ **注意**：这是开发环境的设置！正式发布需要：
- 服务器需要有 HTTPS 证书
- 域名需要在微信公众平台配置为合法域名

## 📝 使用流程

### 更新题库的完整流程：

1. **启动后台服务**（如果没启动）
   ```bash
   cd admin && npm start
   ```

2. **打开后台管理界面**
   - 浏览器访问：`http://localhost:3000`
   - 可以添加、编辑、删除题目

3. **保存修改后，小程序自动同步**
   - 小程序启动时会自动检查更新
   - 如果有新版本会自动下载
   - 下载完成后使用新题库

4. **验证更新**
   - 在小程序控制台查看日志
   - 应该能看到：`🎉 题库已更新！`

## 🎯 功能特点

### ✨ 智能同步

- ✅ 启动时自动检查版本
- ✅ 发现新版本自动下载
- ✅ 本地缓存，离线可用
- ✅ 失败时降级使用本地版本

### 🔄 版本管理

- 使用题库文件修改时间作为版本号
- 每次后台保存题目，版本号自动更新
- 小程序只在有新版本时才下载

### 💾 本地缓存

- 首次使用：下载完整题库并缓存
- 后续使用：优先使用缓存
- 更新时：只在需要时下载新题库

## 🌐 部署到服务器（可选）

如果你想让用户在正式环境也能自动更新，需要：

### 1️⃣ 准备服务器

- 购买云服务器（阿里云、腾讯云等）
- 安装 Node.js
- 配置防火墙，开放 3000 端口

### 2️⃣ 部署后台

```bash
# 在服务器上
git clone 你的项目
cd admin
npm install
npm start  # 或者使用 pm2 等进程管理工具
```

### 3️⃣ 配置 HTTPS

- 申请 SSL 证书（Let's Encrypt 免费）
- 配置 Nginx 反向代理
- 确保能通过 HTTPS 访问

### 4️⃣ 配置微信公众平台

1. 登录微信公众平台
2. 进入「开发」→「开发管理」→「开发设置」
3. 在「服务器域名」→「request合法域名」中添加：
   ```
   https://你的域名
   ```

### 5️⃣ 修改小程序代码

将 `utils/questionManager.js` 中的服务器地址改为：

```javascript
const SERVER_URL = 'https://你的域名'
```

## 🐛 故障排除

### 问题1：小程序无法连接到后台

**检查**：
- 后台是否启动了？
- 服务器地址是否正确？
- 开发者工具是否勾选了「不校验合法域名」？

**解决**：
```bash
# 确认后台运行
curl http://localhost:3000/api/version
# 应该返回版本信息
```

---

### 问题2：题库没有更新

**检查**：
- 后台是否真的保存了题目？
- 查看小程序控制台日志
- 是否看到版本对比信息？

**解决**：
- 确认后台文件已更新（查看 `data/questions.js` 的修改时间）
- 重新保存一次题目
- 重启小程序

---

### 问题3：下载很慢或失败

**原因**：
- 题库太大
- 网络不好

**解决**：
- 保持网络畅通
- 小程序会自动重试，失败了使用本地版本
- 不影响用户使用
